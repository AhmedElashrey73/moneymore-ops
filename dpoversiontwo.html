Consent flows — visual proposals

Two proposed flows for DPO review. Flow 1 covers the Offers tab consent gate with two independent checkboxes. Flow 2 covers the Elgameya promo code reveal and redirect with plain text disclosure.

Flow 1 — Offers tab consent gate

Curiosity gap + two-checkbox consent

The Offers tab is visible in the bottom navigation but locked until the user chooses their preferences. The consent screen presents two independent checkboxes — each separately auditable and revocable from Settings at any time. Note on wording: rather than "I consent to", checkboxes are framed as user preferences. Pending DPO confirmation that affirmative checkbox language satisfies PDPL requirements without the literal phrase "I consent".

Flow 2 — Elgameya promo code and redirect

Consent → promo code reveal → plain text disclosure → redirect

After the four intent questions, the user sees a consent screen with a checkbox before the promo code is revealed. The redirect disclosure appears as plain text below the "Open Elgameya" button — visible before the tap, no pop-up required. Pending DPO confirmation that plain text disclosure satisfies the requirement.
