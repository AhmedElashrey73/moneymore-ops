Offers Tab v1
A curated feed of third-party discount offers. Users browse Available, claim via code reveal or merchant redirect, and review their claimed offers in My Offers.
The problem
Users currently have no place inside MoneyMore to discover deals on the brands they already spend with. Offers gives the app a recurring reason to come back — beyond budgeting and tracking — and creates a low-stakes value moment that works for every segment without targeting. The flow has to feel curated and trustworthy, not adverty: clear terms, simple claim, and a reliable place to find a code again later.

Accessibility — what changes what you draw
  • Touch targets — 44×44px minimum on every interactive element. Includes the segment pills, hero card tap zone, regular card tap zone, sheet close (✕), and the Copy button.
  • Contrast — 4.5:1 minimum for text, 7:1 for the discount amount and revealed code. The hero card's Featured badge gold-on-navy and the urgent expiration pill red-on-navy both need contrast verification.
  • Never colour-only for state. Capped cards need "Claimed" or "Back in X days" text alongside the muted treatment. Disabled CTAs need text or icon. Urgent expiration (red pill) needs the ⏱ icon prefix.
  • Focus order — segment control → hero card → list eyebrow → regular cards top-to-bottom. Inside the bottom sheet — close button → primary CTA → terms text. RTL flips visual direction but focus order stays top-to-bottom.
  • Code value reveal must be selectable text, not an image — for screen readers, accessibility services, and manual copy fallback if the Copy button fails.
