Sign-up & Onboarding (iOS)
A new iOS user signs up via Apple, Google, or phone, sets up automatic transaction tracking through the iOS Shortcuts app, and lands on Home — either in empty-but-armed state (Shortcut on, waiting for first transaction) or manual-only state (Shortcut skipped). A non-blocking re-prompt nudges users who skipped.
The problem
iOS doesn't allow background SMS reading. Automatic tracking on iOS depends on a 9-step setup inside Apple's Shortcuts app, which we don't control. The onboarding has to make that setup feel like 60 seconds of progress. Every Skip path needs to feel safe to take. The Shortcut tutorial is the most important moment of the whole iOS app.

Flow 1 — Welcome → Entry → AutoFill phone → OTP with QuickType code → Name capture
Three welcome screens with a language toggle in the top-right. Then the Entry screen carries three equal-weight auth options: Apple, Google, and phone. When the phone field is focused, iOS may surface the user's own number as an AutoFill suggestion in the QuickType bar above the keyboard — pulled from the user's "My Card" contact. Then a WhatsApp/SMS OTP channel choice with no default, then OTP entry where the incoming code surfaces in the QuickType bar for one-tap fill. Name capture only fires on the phone path — Apple and Google return the user's name from their account.

Flow 2 — Shortcut tutorial · 10 steps
This is the iOS-specific replacement for SMS permission. iOS doesn't allow apps to read SMS in the background — the only way to track bank transactions automatically is to set up an iOS Shortcuts Automation that fires a "Log from SMS" action every time a bank message arrives. The 9 steps walk the user through setting that Automation up inside Apple's Shortcuts app. Step 10 confirms it's done. The whole thing has to feel like 60 seconds of progress, not a 9-step tutorial.

Accessibility — what changes what you draw
  • Touch targets — 44×44pt minimum on every interactive element. Includes the language toggle on welcome screens, OTP boxes, the resend link, the switch-channel link, the "Wrong number?" / back chevron, the "I'll do this later" app-bar action, the Skip link on the tutorial intro, the recovery link below the manual-only Home hero, and the centre [+] button.
  • Contrast — 4.5:1 minimum for text, 7:1 for any financial figures (Home Total amounts, category breakdown amounts).
  • Never colour-only for state. OTP error pairs red border with inline error text and warning icon. Disabled CTAs (OTP channel CTA before selection, Continue when name is empty) need a lock icon, opacity change, or text signal beyond colour. Selected channel state in the OTP channel sheet uses both border colour and a radio dot.
  • Focus order — input → primary CTA → secondary actions → dismiss. On Arabic layouts the visual direction flips to RTL but focus order stays top-to-bottom, not visual-left-to-right.
  • Tutorial step illustrations need text equivalents. Every step screen carries the instruction in copy — the visual is supportive, not the primary information. A user who can't see the mock still knows what to tap.
  • Welcome-back screen must announce the user's name to VoiceOver. Name is the moment-of-recognition signal — not just decoration.
