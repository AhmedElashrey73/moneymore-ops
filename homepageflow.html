Home Tab Revamp
A summary surface, not a dashboard. The Home Tab answers "what's happening with my money this month?" at a glance, and surfaces one next action when one exists.
The problem
Returning users see the same flat dashboard every session with no directional nudge. The revamp composes eight slots across six states into three big cards: Total Expenses (with category breakdown inside), Smart Budget, and Latest Spending. Slots that have nothing to say collapse. The smart budget slot is the one hero; everything else is informational.

Pluralisation on moment 19 (View all): Arabic plural form changes with count. 3–10 budgets use ميزانيات (plural); 11+ revert to ميزانية (singular form) per Arabic counted-noun rules. Implement as a number-aware string resource, not a simple [N] swap.
Accessibility — what changes what you draw
  • Touch targets — 48 dp minimum on every interactive element. Particular attention: header bell and avatar (icons may be visually smaller but tappable area ≥48 dp), contained rows across Total Expenses / Mode B / Latest Spending, sheet field rows, card-internal CTAs.
  • Contrast — 4.5:1 for text, 7:1 for financial figures. Financial figures here: Total Expenses amount (32px tabular-nums), category amounts, Mode B spent/limit text, transaction amounts in cards and in Transaction Detail header, Mode A suggested limits, push notification amounts. Check the 32px Total Expenses amount especially.
  • Budget state conveyed by label or position, not colour alone. Every Mode B row carries a state pill (Under / Near / Over) alongside the bar colour. Over-limit additionally carries the "[over] over" text on the right.
  • Bell badge visible cap "9+" differs from the TalkBack label ("More than 9 unread notifications"). Count-aware label set in moment 4.
  • Focus order top-to-bottom. Home Tab: header → review prompt / offline pill (if present) → Total Expenses card (amount → count → category rows → See all) → Smart Budget card (label → rows/mode-body) → Latest Spending card (label → rows) → Add Transaction. Transaction Detail sheet: close → header → fields in order → actions. Arabic RTL: visual flips at layout level, focus order stays top-to-bottom.
