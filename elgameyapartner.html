Elgameya Partner Offer
A user claims a one-time 300 EGP promo code for their first Elgameya circle installment, then opens the Elgameya app to redeem it. This is MoneyMore's first real partner offer — the design sets the pattern for every partner that comes after.
The problem
The Offers tab today is empty stagecraft. Elgameya is the first partner with real value to give a real user. The flow has to feel like a coupon worth keeping — not a generic banner ad. The moment of code reveal is where the offer earns its place: the code must feel tangible, the redirect must feel intentional, and the user must understand they're leaving MoneyMore without it feeling abrupt.

Accessibility — what changes what you draw
  • Touch targets — 44×44px minimum on every interactive element. Copy button, sheet dismiss ✕, card CTA pill, redirect CTA.
  • Contrast — 4.5:1 minimum for text, 7:1 for the 300 EGP amount in the You save hero. The white-on-purple gradient needs to meet 7:1 on the darker end of the gradient, not just the lighter end.
  • Never colour-only for state. Copied confirmation uses ✓ + "Copied" text, not just a green button. Capped card uses "Fully claimed" stamp + greyed styling, not greyed styling alone.
  • Focus order — sheet head → code → Copy → disclosure → Open Elgameya. The disclosure must sit in the focus order before the redirect CTA, not after. On Arabic RTL, visual flips but focus order stays the same.
  • The code itself must be selectable text, not an image or styled background — screen readers must read out "M M dash E L G dash 7 K 2 P" character by character.
