Prototype Designer brief iOS Draft v3 DB-011 Home Tab — iOS
Home Tab (iOS)
The most-used surface in the app. Three cards stacked — Total Expenses (with top categories), Smart Budget (four behaviour modes), Latest Spending (horizontal carousel). A Shortcuts setup card earns its place at the top until tracking is on. Universal across salary workers, freelancers, and students.
The problem
Users open the Home Tab to answer one question fast — am I doing okay this month? The surface has to give that answer in under two seconds for a returning user, while still earning the first tap from a new user who has nothing on it yet. Smart Budget shifts behaviour across four modes (recommendations, active tracking, hybrid, generic) without ever changing its slot or title. The Shortcuts setup card is the gateway to passive tracking; if it doesn't get tapped, nothing else on this screen has data to work with.

Accessibility — what changes what you draw
  • Touch targets — 44×44pt minimum on every interactive element. Includes the Mode A ✓ and ✕ buttons (currently 30pt visual — pad the tap target outward), the Mode B row (whole row tappable, not just the icon), the centre [+] button, partial-sheet field rows, and dismiss links.
  • Contrast — 4.5:1 minimum for text, 7:1 for financial figures. Total Expenses amount, budget spend/cap numbers, transaction card amounts, and "EGP left / over" all need the higher ratio. Check the Shortcuts setup card body copy on its navy gradient — the rgba(255,255,255,0.82) lands close to the floor.
  • Never colour-only for state. Smart Budget rows ship the status pill (text + colour) before the bar colour. On-track rows use no pill — clean baseline. Recommendation rows use the dashed border in addition to the green ✓ to signal "suggested, not active".
  • Focus order — vertical top-to-bottom, card by card. Inside Smart Budget, focus walks each row's whole touch zone, then the row's ✓ and ✕ if present, before moving to the footer CTA. On Arabic layouts the visual direction flips to RTL but focus order stays top-to-bottom.
  • Latest Spending carousel — horizontal scroll needs both swipe and a "See all transactions" path. Carousel should announce its position to VoiceOver ("transaction 1 of 5"). Keyboard / switch-control users get to "See all" via the section header link.
  • Dynamic Type up to XXL. Smart Budget rows and Total Expenses category rows must reflow without truncating the spend/cap fraction; icon + name can stack above the amount on the largest sizes rather than ellipsing. Carousel cards grow vertically; horizontal width fixed.
