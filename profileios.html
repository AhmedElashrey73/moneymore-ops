Profile (iOS)
A single Profile screen for managing account info, preferences, permissions, automatic tracking setup, support, and account actions. iOS counterpart to the Android Profile, with iOS-native patterns (action sheets, alerts, settings deep-link) and iOS-only divergences (Theme = System / Dark / Light, Automatic tracking via Shortcuts).
The problem
Activated iOS users need a single place to manage their identity, language, theme, permissions, automatic tracking setup, and account actions — without contacting support. The screen carries weight: it's where users add a backup sign-in method, recover from denied push permissions, set up the iOS Shortcut that drives automatic tracking, and ultimately sign out or delete their account. The design has to be calm, scannable, and let destructive actions feel weighty without feeling buried.

Accessibility — what changes what you draw
  • Touch targets — 44×44px minimum on every interactive element. Includes the gear icon in the hero, status pills (which are tappable to enter the relevant flow), the dashed Add row's tap area, the back chevron, and the SSO buttons.
  • Contrast — 4.5:1 minimum for text, 7:1 for status pills (Granted / Denied / On / Not set up). The hero's white-on-navy passes; the rgba transparencies on hero subtext need a contrast check at the gradient's lightest point.
  • Never colour-only for state. Push Denied row pairs red tint with explicit "Denied" pill text and a recovery-focused subtext. Disabled CTAs (Continue on Step 1, Send on empty Contact form) need text or opacity signal beyond the grey background.
  • Focus order — hero gear → first row of each section, top-to-bottom. On Arabic layouts the visual direction flips to RTL but VoiceOver focus order stays top-to-bottom.
  • Avatar VoiceOver label is not character-limited. Use the descriptive form "Profile photo, initials shown" — VoiceOver reads full sentences naturally.
  • Status pill VoiceOver labels read as "{permission}, status {state}" — e.g. "Push notifications, status Denied. Tap to enable in Settings." Don't rely on the visual pill alone.
  • Destructive actions get explicit VoiceOver hints — "Yes, delete it" reads "Deletes your account permanently. Double tap to confirm." The verbal asymmetry mirrors the visual asymmetry between the two CTAs.
