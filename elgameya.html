Targeting logic

Who sees the Elgameya offer

Not all MoneyMore users are equally relevant for Elgameya. The ideal audience is users who have already demonstrated savings intent through the product — they are the most likely to convert and the most valuable leads for Elgameya.

Primary and secondary target segments
Primary segment
Users who have set a goal or created a budget
These users have already demonstrated financial intentionality inside MoneyMore. They have a savings purpose — which is exactly what a gameya serves. They are the highest-intent audience and the strongest leads for Elgameya. Target this segment first via push notification and in-app card placement.
Secondary segment
Active users with regular transaction tracking
Users who are actively engaged with their transactions but have not yet set a goal or budget. They are financially aware and open to savings tools. Surface the Elgameya card in the Offers tab for this segment — let curiosity drive the entry rather than a direct push.
Promo code proposal

Unique codes to prevent abuse

A single shared promo code for all MoneyMore users is an abuse risk — it leaks, gets shared outside the platform, and makes performance tracking impossible. We propose unique per-user codes. Three options below — we would like to align on the preferred approach.

Three approaches to unique promo codes
01
Pre-generated code pool — recommended starting point
Elgameya provides MoneyMore with a batch of unique single-use codes. MoneyMore assigns one per user at the moment they tap "Get my promo code". Each code is used once and marked as redeemed. No API integration needed. Elgameya replenishes the pool as needed. Simple, fast to implement, zero technical dependency between the two systems.
Recommended — lowest technical lift
02
Real-time API code generation
When the user taps to get their code, MoneyMore calls Elgameya's API in real time and a unique code is generated and returned instantly. Fully automated, no pool management, scales infinitely. Requires API integration between MoneyMore and Elgameya before launch.
Recommended long-term
03
Structured code with embedded identifier
A code format such as MM-[UserID]-ELG that encodes the MoneyMore user reference. Elgameya validates on their side that the code maps to a legitimate MoneyMore referral. Simple but exposes a user identifier in the code itself — minor privacy consideration to confirm with DPO.
